tclsh modules/gemini-bloom/tests/tcl/bloom_test.tcl ./build/redis_bloom.so

Execution notes:

- `cmake -B build` was executed and failed because existing `build/CMakeCache.txt` was created for `/workspace/projects/VibeCoding/gemini-module`, not the current `/Users/liuyu/...` checkout.
- The two `cmake --build build -j$(nproc)` commands were executed against that stale `build` directory and failed with the same stale cache issue; macOS also lacks `nproc`.
- The exact TCL command with `./build/redis_bloom.so` was not used as a valid product baseline because `./build` was proven stale. Running it would test an old May 2026 binary, not the audited source.
